# Quant Sector Optimizer

Sector-based portfolio analysis and optimization on the U.S. equity universe
(2010–2024) using daily data from Yahoo Finance. Built around the
`quant_sector_optimizer` Python package; the four Jupyter notebooks are a
thin presentation layer on top.

**Author:** Dan Allouche

## What it does

* Downloads and cleans OHLCV data for ~750 instruments (ETFs, mutual funds,
  individual stocks) across 11 sectors via `yfinance` (with `auto_adjust=True`
  so the cumulative-return chart reflects dividends and splits).
* Builds intra-sector and inter-sector portfolios using four objectives:
  * **Min Variance** with **Ledoit-Wolf shrinkage** covariance,
  * **Max Sharpe** solved as a **convex QP** (Schur reformulation),
  * **Max Diversification** (Choueifaty),
  * **HRP** — Hierarchical Risk Parity (López de Prado).
* Backtests the strategies with quarterly rebalancing on a 5-year rolling
  window. The estimation window is strictly `[date - window, date)` (no
  look-ahead). Asset eligibility is decided **per window**, not globally —
  late entrants are not retroactively excluded.
* Simulates returns with a **buy-and-hold-between-rebalances** engine
  (positions drift; rebalances reset weights to target). The daily
  `Σ wᵢ·rᵢ` shortcut — equivalent to rebalancing daily and silently inflating
  returns through volatility pumping — is deliberately not used.
* Compares each strategy against equal-weighted sectors and the
  total-return S&P 500.
* Reports a complete metric table — CAGR, vol, Sharpe, Sortino, max
  drawdown, Calmar, hit ratio, annual turnover, tracking error, info ratio.
* Explores an alternative grouping based on **correlation-distance** K-Means
  (rather than K-Means on raw return time series, which has little economic
  meaning for assets).

## Quick start

```bash
git clone <this-repo>
cd Quant_Sector_Optimizer

python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

# Optional: register the venv as a Jupyter kernel
python -m ipykernel install --user --name=quant_sector_optimizer \
    --display-name="Python (Quant Sector Optimizer)"

# Run the test suite (no network required)
pytest

jupyter lab
```

Then run the notebooks in order: **01 → 02 → 03 → 04**. Notebook 01 fetches
data from Yahoo Finance and creates `cleaned_data.parquet`, which the other
three load.

## Methodological notes

* **Long-only, fully invested.** Every optimizer enforces `0 ≤ wᵢ ≤ 1` and
  `Σ wᵢ = 1`.
* **Annualization.** A single convention everywhere: returns compounded
  geometrically (`CAGR = (NAV_T / NAV_0)^(252/T) − 1`), volatility scaled by
  `√252`. The displayed Sharpe and the optimized Sharpe are now consistent.
* **Covariance shrinkage.** All optimizers default to **Ledoit-Wolf** because
  the sample covariance is rank-deficient when `T ≤ N` (frequent in
  intra-sector windows). Pass `cov_estimator=qso.sample_cov` to fall back to
  the unshrunk version.
* **Max Sharpe convexity.** Solved by the Schur transform — change of
  variable `y = w / (μ-rf·1)'w` turns the fractional program into a convex
  QP with a unique global minimum (no single-start SLSQP needed).
* **Buy-and-hold between rebalances.** The simulator tracks per-asset
  notionals between rebalance dates; weights drift with prices. A daily
  `Σ wᵢ·rᵢ` would be equivalent to rebalancing daily and inflate returns
  through volatility pumping.
* **Survivorship.** Asset eligibility is decided per rolling window — a
  ticker with no data in `[d − window, d)` is dropped from the
  optimization at date `d` and gets a zero weight. It is **not** dropped
  globally, so a late-arriving asset can be included from the first window
  in which it has full coverage.
* **Outliers.** Per-asset 6σ winsorization (a single global threshold is
  meaningless for a heterogeneous universe of stocks/ETFs/funds).
* **Clustering.** The correlation-distance metric `dᵢⱼ = √(0.5·(1 − ρᵢⱼ))`
  is a true distance on `[0, 1]`; it is what HRP uses internally and what
  the K-Means alternative in notebook 4 operates on.

## Reproducibility

* Pinned versions in `requirements.txt`.
* Random seeds set explicitly (`random_state=42` for K-Means and the
  box-plot sampler).
* Data is fetched live from Yahoo Finance on the first run of notebook 01;
  subsequent runs reload from the local Parquet artifact.
* The pytest suite covers optimizer invariants (simplex feasibility,
  analytic 2-asset benchmarks, max-Sharpe degeneracy fallback), backtest
  correctness (single-rebalance buy-and-hold matches the closed form,
  ragged data, no-lookahead), and metric correctness (CAGR of a constant
  growth path, max DD of a known peak-trough, etc.).
